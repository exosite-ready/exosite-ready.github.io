If that assert failed that means that no network device has been registered; and the possible causes are

  • you don't have UIP built (CONFIG_HAS_UIP = n)
  • you did register any network device, that should be done in the network stack adaptation layer

If this assert fails then either

  • the SSL support is not built in
  • there was an error during the initalization of the SSL library
