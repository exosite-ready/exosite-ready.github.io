void(* protocol_client_ops::close) (struct protocol_client_class *self)

Close the protocol client context.

A protocol client implementation might need some cleanup; e.g there can be internal queues, etc that need to be cleaned up between socket context changes E.g this can as a separator between requests if the request/response was not successful Otherwise it can happen that a previous request is retransmitted and we get a response to that

Returns
ERR_SUCCESS on success ERR_* error code otherwise

int32_t(* protocol_client_ops::recv_response) (struct protocol_client_class *self, int32_t *response_code, int32_t *message_id, char *data, size_t max_len, size_t *payload_size)

Try to read a response to the previous request.

  • It returns ERR_SUCCESS if it successfully read and parsed a response. In that case it also sets the response code, the message id copies the payload to data and sets the payload size.
  • It returns ERR_WOULD_BLOCK if it could not read a response.
  • It returns other ERR_ error code if it could read a response but it could not parse it
Parameters
[in]self- the protocol object (HTTP or COAP currently)
[out]responsecode The response code
[out]message_idThe token that identifies the request - response pair
[out]dataThe response itself in null terminated format (c string)
[in]lenThe length of the ouput buffer
Returns
ERR_SUCCESS on success ERR_* error code otherwise
